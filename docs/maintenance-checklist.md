# 保守チェックリスト

講義ノートを更新・公開するときの確認項目です．
大きな変更だけでなく，小さな修正でも公開前に一通り確認します．

## 編集前

- [ ] `git status --short` で作業状態を確認した
- [ ] 更新対象の章，データ，設定ファイルを確認した
- [ ] 公開情報と非公開情報の置き場所を確認した
- [ ] 管理者用メモは `private/` に置くことを確認した

## 編集後

- [ ] 意図したファイルだけが変更されている
- [ ] `requirements.txt` と本文の依存パッケージ説明が一致している
- [ ] Jupyter Notebook の出力や表示が不自然に壊れていない
- [ ] Markdown の見出しレベルが不自然に飛んでいない
- [ ] 公開してはいけない情報が `README.md` や `docs/` に入っていない

## ビルド確認

- [ ] `jb build --all .` が成功する
- [ ] Jupyter Book の警告が残っていない
- [ ] `_build/html/index.html` をローカルで確認した
- [ ] `.ipynb` と `.md` のダウンロードリンクを確認した
- [ ] `docs/` と `private/` が Jupyter Book の HTML に含まれていない

## Git 確認

- [ ] `_build/` が Git 管理対象に入っていない
- [ ] `private/` が ignored になっている
- [ ] `git status --short --ignored private` で `!! private/` と表示される
- [ ] commit message が変更内容を表している

## 公開後

- [ ] GitHub Pages のページが更新されている
- [ ] 主要ページがブラウザで表示できる
- [ ] ダウンロードリンクが期待通り動く
- [ ] 公開用変更履歴を必要に応じて更新した
- [ ] 管理者用作業記録を必要に応じて `private/` に残した
